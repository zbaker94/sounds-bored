## Task 2: Symlink Vulnerability Fixes (symlink-vulnerabilities.md)

**Review file:** `docs/review-v1.5.4/symlink-vulnerabilities.md`
**Findings:** SEC-1, SEC-3

### SEC-3 — Reject symlink root before WalkDir

- [ ] **Step 1: Add root symlink check before `WalkDir::new`**

  In the async task body, before the `for entry in walkdir::WalkDir::new(&source_path)` loop (~line 507), add:

  ```rust
  // Reject symlink roots — walkdir follows the target when root is a symlink.
  let root_meta = std::fs::symlink_metadata(&source_path)
      .map_err(|e| format!("Cannot stat source path: {}", e))?;
  if root_meta.file_type().is_symlink() {
      return Err("Source project path must not be a symlink".to_string());
  }
  ```

### SEC-1 — Close TOCTOU gap for `extra_sound_paths`

- [ ] **Step 2: Open file handles during validation, reuse in async task**

  In the synchronous validation loop (~lines 459–464), instead of only checking metadata, collect open `File` handles:

  ```rust
  let mut extra_files: Vec<(String, std::fs::File)> = Vec::new();
  for (i, extra) in extra_sound_paths.iter().enumerate() {
      if !is_allowed_audio_extension(extra) {
          return Err(format!(
              "extra_sound_paths[{}] must be an audio file", i
          ));
      }
      let meta = std::fs::symlink_metadata(extra)
          .map_err(|e| format!("extra_sound_paths[{}]: {}", i, e))?;
      if !meta.file_type().is_file() {
          return Err(format!("extra_sound_paths[{}] must be a regular file", i));
      }
      // Open during validation — prevents TOCTOU replacement with a symlink
      // between validation and the async File::open below.
      let f = std::fs::File::open(extra)
          .map_err(|e| format!("extra_sound_paths[{}]: cannot open: {}", i, e))?;
      extra_files.push((extra.clone(), f));
  }
  ```

  Then update the async "Add extra sounds" loop to use these handles instead of re-opening.
